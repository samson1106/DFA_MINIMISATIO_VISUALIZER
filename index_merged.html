What is a DFA?

A Deterministic Finite Automaton is a mathematical model of computation — like a tiny robot that reads an input string one character at a time, jumping between states, and decides at the end: accept or reject.

🤖

What is a DFA?

The 5-tuple formal definition, states, alphabet, transitions, start and accept states explained simply.

A DFA (Deterministic Finite Automaton) is formally defined as a 5-tuple:

M = (Q, Σ, δ, q₀, F)

Q — A finite set of states. Think of them as rooms in a building.

Σ — The alphabet, a finite set of input symbols (e.g., {0, 1}).

δ: Q × Σ → Q — The transition function. Given a state and a symbol, it tells you which state to go to next. Always exactly one destination — that's the "deterministic" part.

q₀ ∈ Q — The start state, where the machine begins reading.

F ⊆ Q — The set of final (accepting) states. If the machine ends here after reading all input, it accepts the string.

The DFA accepts a string w if and only if the computation starting at q₀ ends in a state in F.

⚡

Why Minimize?

Removing redundant states makes DFAs faster, smaller, and proves uniqueness — the minimized DFA is canonical.

Different DFAs can recognize the same language but have different numbers of states. Minimization finds the unique smallest DFA for a given regular language.

Why does this matter?

• Efficiency — Fewer states means less memory and faster execution in hardware or software implementations.

• Canonicality — The minimized DFA is the unique representative of its language. Great for testing DFA equivalence.

• Regex Engines — Browsers, text editors, and compilers use minimized automata under the hood for pattern matching.

• Hardware Design — Digital circuit controllers are often modeled as DFAs. Fewer states = fewer gates = cheaper chips.

The Myhill–Nerode theorem guarantees: for any regular language, the minimized DFA is unique up to state renaming.

📊

Table-Filling Algorithm

The Myhill–Nerode table-filling method — marking distinguishable pairs and merging equivalent ones.

Two states p and q are distinguishable if there exists some string w such that exactly one of δ*(p,w) and δ*(q,w) is a final state.

States that are not distinguishable are called equivalent — they can be safely merged.

Step 1: Mark (p, q) if one is final, other is not.
Step 2: Mark (p, q) if ∃ symbol a: (δ(p,a), δ(q,a)) is marked.
Step 3: Repeat until no new marks. Merge unmarked pairs.

The algorithm runs in O(n² · |Σ|) time. After convergence, each equivalence class becomes one state in the minimized DFA.

🔀

Partition Refinement

An alternative view: start by splitting states into Accept vs Non-Accept groups, then keep splitting until stable.

Partition refinement starts with the coarsest possible partition: {F} (accepting states) vs {Q\F} (non-accepting states).

P₀ = { F, Q∖F }
Pₙ₊₁ = split Pₙ until stable

Splitting rule: A group G is split if there exist states p, q ∈ G and symbol a such that δ(p, a) and δ(q, a) land in different groups.

States in the same final group are equivalent — they behave identically for all possible future inputs.

🔗

State Merging

How equivalent states collapse into one — constructing the minimized DFA from equivalence classes.

Once we have the final equivalence classes, we build the minimized DFA:

States: one representative per equivalence class
Start: class containing original q₀
Final: classes containing any original final state
δ_min([q], a) = [δ(q, a)]

Where [q] denotes the equivalence class of state q. The transition is well-defined because equivalent states always transition to states in the same class.

Dead states (unreachable states) are removed before minimization, as they cannot affect the language accepted.

🎓

Worked Example

Follow a complete example from scratch — a DFA over {0,1} that accepts strings ending in "01".

DFA accepting strings over {0,1} ending in "01":

States: q0, q1, q2, q3   Alphabet: {0, 1}
Start: q0   Final: {q2}

δ(q0, 0) = q1    δ(q0, 1) = q0
δ(q1, 0) = q1    δ(q1, 1) = q2
δ(q2, 0) = q3    δ(q2, 1) = q0
δ(q3, 0) = q1    δ(q3, 1) = q2

Minimization: Check distinguishability of all pairs:

• q0 vs q2: q2 is final, q0 is not → marked ✕

• q1 vs q3: δ(q1,1)=q2, δ(q3,1)=q2 (same); δ(q1,0)=q1, δ(q3,0)=q1 (same) → equivalent ≡

So q1 and q3 merge! The minimized DFA has only 3 states.

The Minimization Algorithm — Step by Step

1

Remove Unreachable States

Run BFS/DFS from the start state. Any state not visited is unreachable and can be safely deleted — it doesn't contribute to the language.

2

Initialize the Pair Table

Create a triangular table of all pairs (p, q) where p ≠ q. Mark any pair where one state is final and the other is not — these are immediately distinguishable.

3

Propagate the Marks

For each unmarked pair (p, q): if there exists a symbol a such that (δ(p,a), δ(q,a)) is already marked, then mark (p, q) too. Repeat until no new marks in a full pass.

4

Identify Equivalence Classes

All remaining unmarked pairs are equivalent. Use union-find to group them into equivalence classes — each class becomes one state in the minimized DFA.

5

Build the Minimized DFA

Create one state per equivalence class. The start state is the class of q₀. A class is final if it contains any original final state. Build transitions using any representative of each class.

📐 Theory Quick Reference

A DFA is a 5-tuple M = (Q, Σ, δ, q₀, F) — a finite set of states Q, an alphabet Σ, a total transition function δ: Q × Σ → Q, a start state q₀, and final states F. It reads input left-to-right, one symbol at a time, deterministically moving between states.

Accepts w iff δ*(q₀, w) ∈ F

Multiple DFAs can accept the same language but differ in state count. Minimization finds the unique smallest equivalent DFA. By the Myhill–Nerode theorem, this minimal DFA is unique up to state renaming — it is the canonical form of the language.

Fewer states → faster matching · smaller circuits · canonical form

States p, q are distinguishable if ∃ string w s.t. exactly one of δ*(p,w), δ*(q,w) is final. The algorithm marks all such pairs iteratively; remaining unmarked pairs are equivalent and can be merged.

Step 1: Mark (p,q) if one is final, other is not. Step 2: Mark (p,q) if ∃a : (δ(p,a), δ(q,a)) is marked. Step 3: Repeat until stable. Merge unmarked pairs.

Time complexity: O(n² · |Σ|)

Start with the coarsest partition {F, Q∖F}. Repeatedly split any group whose members transition to different groups on some symbol. Stop when no group can be split — each final group is one equivalence class.

π₀ = {F, Q∖F} πₙ₊₁ = refine πₙ until πₙ₊₁ = πₙ (stable)

Produces the same result as Myhill-Nerode. More intuitive visually.

Choose Minimization Method

Myhill-Nerode (Table-Filling) CLASSIC
Builds a distinguishability table for all state pairs. Iteratively marks distinguishable pairs until convergence. Remaining unmarked pairs are equivalent and merged.

🎓 Beginner tip: Think of it like a checklist — you cross off pairs of states that are "different", and whatever is left uncrossed can be combined.
Grouping Method (Partition Refinement) INTUITIVE
Starts by splitting all states into two groups: accepting and non-accepting. Repeatedly refines groups until no further splits are possible. Each final group becomes one state.

🎓 Beginner tip: Think of it like sorting cards into piles — you keep dividing piles until every card in a pile is truly identical in behavior.
💡 Not sure which to pick? Both algorithms always produce the same minimized DFA. Myhill-Nerode is the classic textbook approach and great for understanding the theory. Grouping Method is more visual and easier to follow step-by-step. For beginners, we recommend the Grouping Method first!
